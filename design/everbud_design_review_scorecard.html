Everbud — Design Review

Landing page de marketing · everbud.vercel.app

Primera impresión
El sitio comunica app de nicho con personalidad propia. Lo primero que ve el ojo es el heading verde+negro, luego los CTAs verdes, luego el vacío enorme antes de las features. Las áreas sin propósito claro son: el espacio blanco masivo entre el hero y la sección de features (¿faltó un mockup de producto?) y los pasos de "¿Cómo funciona?" que flotan sin ancla visual. Una palabra: incompleto.

⚡ Quick Wins (impacto alto, <30 min)
ALTO FINDING-001: Hero sin imagen de producto

El espacio vacío entre el CTA y la sección de features hace que el sitio parezca sin terminar. Un usuario que llega no tiene evidencia visual de cómo es el producto. La confianza de conversión cae.

Fix: agrega un mockup del dashboard entre el hero y las features:

/* Hero wrapper con imagen abajo */ .hero-section { padding-bottom: 0; } .hero-product-preview { margin: 40px auto 0; max-width: 860px; border-radius: 12px 12px 0 0; overflow: hidden; box-shadow: 0 -4px 32px rgba(0,0,0,0.08); /* screenshot del dashboard real aquí */ }

Impacto: comunica credibilidad, reduce ansiedad de "¿cómo funciona esto?", mejora conversión CTR.

ALTO FINDING-002: Reemplazar emojis con íconos SVG reales

Los emojis (🌿💀📋💧) tienen colores inconsistentes con la paleta, rendering diferente en cada OS y comunican "hecho en 20 min". El skull (💀) para "Registro de mortalidad" es objetivamente la peor decisión de diseño en el sitio.

/* Sustituir con SVG monocromo en tu verde primario */ .feature-icon { width: 40px; height: 40px; background: #F0FAF0; /* verde muy claro */ border-radius: 10px; display: flex; align-items: center; justify-content: center; color: #22C55E; /* tu verde */ font-size: 20px; /* para íconos Lucide/Tabler */ }

Impacto: paleta coherente, apariencia profesional, sin sorpresas de rendering cross-platform.

ALTO FINDING-003: Fondo plano viola la Landing Page Rule

El hero sobre fondo cream (#F5F3EE) sin ninguna variación no ancla la sección. Las cards blancas tampoco se separan bien del fondo.

/* Hero con gradiente sutil o patrón */ .hero-section { background: linear-gradient( 180deg, #F0FAF0 0%, /* verde muy suave arriba */ #F8F7F2 60%, /* cream normal */ #FFFFFF 100% /* blanco al llegar a features */ ); } /* O: grid pattern sutil */ .hero-section { background-color: #F8F7F2; background-image: radial-gradient(#22C55E18 1px, transparent 1px); background-size: 24px 24px; }
MEDIO FINDING-004: Tipografía display genérica

Inter (o similar) como fuente de display da la señal más clara de "generado por template". Un jardin/naturaleza app puede usar algo con más personalidad para headings sin sacrificar legibilidad.

/* Google Fonts: Fraunces (display serif con personalidad orgánica) */ @import url('https://fonts.googleapis.com/css2?family=Fraunces:ital,wght@0,700;1,700&family=Inter:wght@400;500&display=swap'); h1, h2 { font-family: 'Fraunces', serif; /* solo headings */ } body, p, button { font-family: 'Inter', sans-serif; /* cuerpo sin cambios */ }

Impacto: personalidad de marca inmediata, rompe el slop pattern #11, sin sacrificar legibilidad del cuerpo.
